# 機材構成

## 推奨割り当て

| 機材 | 役割 |
|---|---|
| Mac mini M4 | メイン描画、プロジェクター出力 |
| ノートPC 1 | Goサーバー、運営ダッシュボード |
| ノートPC 2 | 受付、到着チェックイン、演出操作 |
| ノートPC 3 | バックアップ描画、ログ監視、開発 |
| Raspberry Pi 4 Model B | 死活監視、NFC、LED・照明・音響制御候補 |
| プロジェクター | メイン3DCG投影 |

GPU性能の高いノートPCがある場合は、その端末をメイン描画に割り当て、Mac miniをサーバーまたはバックアップに変更する。

## ネットワーク方針

- 参加者向けWebアプリはHTTPSで公開する
- 展示PCは外向きWebSocketで接続する
- 通信断時は最後の状態または自律アニメーションで展示を継続する
- Raspberry Piを大人数用アクセスポイントの中心にはしない
- 設営前に学内ファイアウォールとWi-Fi利用条件を確認する

## 障害時のフォールバック

- メイン描画停止: バックアップ端末で簡易展示を起動
- インターネット切断: 保存済み軌跡を使った生成映像へ切り替え
- 到着判定停止: 受付端末から匿名IDを手動選択
- GPS精度低下: QRチェックポイントを利用
