# ロードマップ

## Phase 1: 技術検証

- [ ] キャンパス内で位置取得精度を実測する
- [ ] UnityとTouchDesignerで同一データを簡易描画する
- [ ] 匿名セッションと位置データモデルを決定する
- [ ] キャンパス座標から3D座標への変換を試す

完了条件: 実際のルートを記録し、簡易3D空間で再生できる。

## Phase 2: MVP

- [ ] Next.js参加画面を作る
- [ ] Go位置受信APIを作る
- [ ] WebSocketで現在位置を配信する
- [ ] 簡易キャンパス上に複数参加者を描画する
- [ ] QR到着チェックインを作る
- [ ] 軌跡を10〜20秒で再生する

完了条件: QR参加から展示場到着までを一台のスマホで通せる。

## Phase 3: 展示統合

- [ ] 本番3D空間と視覚表現を制作する
- [ ] カメラ、音響、到着シーケンスを統合する
- [ ] 運営ダッシュボードを作る
- [ ] 障害時のフォールバックを用意する
- [ ] 機材を本番構成で連続稼働させる

完了条件: 運営者が状態を把握しながら、長時間安定して展示できる。

## Phase 4: 現地試験・公開

- [ ] 主要導線でE2Eテストを行う
- [ ] 複数端末の同時接続試験を行う
- [ ] 複数人同時到着を試す
- [ ] 同意、停止、削除の流れを確認する
- [ ] 設営・リハーサルを行う

完了条件: 受け入れ条件を満たし、展示運用手順が確定している。

## 未決事項

- 展示日と制作期限
- 想定同時参加人数
- 描画エンジンの最終選択
- 3Dモデルの制作方法
- 音響・照明の連携範囲
- 学内ネットワークと位置情報利用の許可
